## Recurring patterns (durable decisions)

The sprint applied the same handful of moves over and over. These are the patterns to recognise and keep:

1. **Deep modules over smeared rituals.** The dominant move: behaviour copy-pasted across N call sites (or buried behind a Hono `Context` + a live D1 fetch) was concentrated behind one named seam, with the _pure decision_ split out as an IO-free, unit-testable function and the route/loader/component reduced to thin glue. This mirrors the existing `src/lib/ingest.ts` deep-module precedent and was applied to ingest, artifacts, the reporter, auth/scope, realtime, analytics, config, provisioning, and the seed/dev-server scripts.

2. **Brand-enforced tenant scope, now at _consumption_.** The single highest-severity finding: `AuthorizedProjectId`/`AuthorizedTeamId` (`src/lib/scope.ts`) were only load-bearing at id _construction_ — every run-scoped query still re-derived the `projectId` predicate by hand, so CLAUDE.md's "impossible to forget" claim was only half true. `makeTenantScope(parts)` is now the single sanctioned launder for the two `as Authorized*Id` casts, and `runScopeWhere(scope)` / `runByIdWhere(scope, runId)` take a `TenantScope` (not raw strings) so a query _cannot_ be scoped from an un-auth-checked id without a type error. The realtime analogue is `authorizeTopicSubscription` — the one isolation check that can't route through a branded id because the WebSocket handshake hands over a raw topic string.

3. **Single source of truth for contracts that were hand-kept-in-sync.** Repeatedly, one fact was encoded in two+ places kept honest only by discipline; each was collapsed to one owner: the **status taxonomy** (`STATUS` registry; `STATUS_BUCKET_MEMBERS` for JS-delta↔SQL-recompute parity), the **theme contract** (`theme.ts`, baked into the FOUC script via `JSON.stringify`), the **wire contract** (`PROTOCOL_VERSION` / `WRIGHTFUL_VERSION_HEADER` exported; response-side Zod schemas; key-set-equality canaries), the **v3 payload builder** (`payload.ts`), **credential crypto** (`token-crypto.ts`), **chunk column-counts** (`chunkInsertRows` derives from row shape), and the **`db.batch` cast** (`runBatch`).

4. **The "mock-the-D1-boundary" test idiom + a standing real-D1 harness gap.** The dashboard vitest harness aliases `void/db` to a throwing stub — `better-sqlite3`'s Drizzle driver has no `batch`, and `db.batch` _is_ the pipeline's atomicity boundary, so DB-issuing functions can't run in unit tests. The strategy throughout: extract the pure decision and unit-test that; assert SQL builders at the fragment / byte-identical level; mock `void/db` (builders as controllable thenables, `db.batch` as a spy) and `@/live` at the boundary to drive orchestration glue. **Every cluster notes the same residual integration gap** — the live D1/R2 round-trip is exercised only by the e2e dogfood suite. A real-D1 unit harness was repeatedly scoped and remains **unbuilt**.

5. **New numbered migrations only.** Schema changes went into fresh numbered migrations, never editing `0000_init` or any applied file (the durable frozen-migrations rule held even though nothing has deployed). Where `void db generate` emitted invalid DDL (it mis-quoted a `COALESCE(...)` expression, splitting it on the comma), the `.sql` was hand-corrected and a unit test guards the bogus form from regressing.

6. **Verifier-narrowing discipline.** The sprint ran finder → verifier on each finding, and the verifier frequently narrowed, merged, or rejected findings _during implementation_. Recorded so future readers don't treat every F-number as an independent change: F16 ⊂ F02; F72/F54/F37 ⊂ F56; F67 already covered by F01; F77 already done by F20; F21 **rejected** (the two `"progress"` literals are an envelope discriminant vs an SSE event type — not a duplicated value); F82 **downgraded** from "broadcast race" to a wasted-round-trip efficiency nit (the broadcast reads status back from the row post-flip, so a race broadcasts the _correct_ terminal); F44/F46/F68 narrowed to test-coverage gaps; F86 narrowed away from a spec-table reshuffle.

7. **Honest scoping & severity.** Several entries flag themselves as shallow DRY cleanups (`sha40`) or robustness fixes for single-caller failure modes (the watchdog `F81`/`F82`) rather than depth/leverage wins, and rate severity **medium** because the system isn't deployed yet. Dead controls were _removed_ rather than half-wired (e.g. the non-functional search input on the flaky page).

## 2026-05-25 — Post-migration feature & polish pass

_(paths shown as authored under `packages/dashboard-void/…`; now at `apps/dashboard/…`.)_

**Settings redesign.** Ported the Settings surface to the Wrightful design handoff. Profile became four client cards over the Better Auth client (`authClient.updateUser`/`unlinkAccount`/`changePassword`/`signOut`) — Identity, Connected accounts (GitHub link/unlink, rendered only when `AUTH_GITHUB_*` configured), Password (only for email/password signups), Sign-out. The team mega-page split into three sub-pages — **general** (name/slug + Danger zone), **members** (list + invite create/revoke), **projects** (list + API-keys deep link) — with `/settings/teams/[teamSlug]` now 302-ing to `…/general`. A new `settings-primitives.tsx` (`SettingsPage`/`Header`/`Card`/`Field`) translated the prototype literally onto the existing Tailwind tokens. `SettingsNav` was rewritten to a grouped Account → {team} → {project} pattern, deriving the active team by _parsing the path_ (no extra DB hit). Profile reads `hasPassword`/GitHub-link via raw SQL against the void-owned `account` table (same pattern as `authz.ts`); actions were moved verbatim from the deleted `index.server.ts`. Out of scope: member-role editing/removal, the "Default Playwright project" card (no schema backing).

**Settings-loader helper dedup.** Before the redesign multiplied them, three near-identical local helpers across sibling `.server.ts` files were lifted into shared modules: `src/lib/slug.ts` (`SLUG_RE`, `SLUG_ERROR`, `isValidSlug`), `src/lib/initials.ts`, and `src/lib/settings-scope.ts` (`redirectWithParam`, `requireOwnerScope`, `requireOwnedProjectScope`). The scope helpers take a `hereFor` builder param so each call site declares its own redirect-back URL while staying deduped, and keep the existing **404-not-403** behaviour (don't leak existence to non-owners). `keys.server.ts`'s loader trims `OwnedProject` back to its pre-refactor shape so the `InferProps` contract with `keys.tsx` is unchanged. _(This `settings-scope.ts` seam was deepened further on 05-30 — see the read-API cluster.)_

**Workspace-selection cookie.** Moved workspace selection off the `userState` DB table into a `wf_workspace` cookie so nothing on the hot path hits the DB for selection: `middleware/01.context.ts` reads the cookie, lets the URL override it, and rewrites it when the URL pins a new team/project. The sidebar was made structurally identical in app and settings modes (switcher always at top, one utility link at the bottom — "Settings" vs "Back to app"); previously the switcher vanished on `/settings/*`. The `shared` bundle fields were renamed `activeTeam/activeProject → selectedTeam/selectedProject` to distinguish them from the URL-bound, auth-checked, branded `c.var.activeProject`. **The cookie is non-authoritative for authorization** — membership is filtered server-side via `resolveTenantBundleForUser`, so a forged/stale cookie resolves only to nulls. Deleted `routes/api/user/last-{team,project}.ts` and `src/lib/user-state.ts` (the switcher's explicit writes are now redundant — middleware writes the cookie next request).

**Reporter PR comment.** Opt-in `postPrComment` reporter option. When enabled and the run finishes in a GitHub Actions PR workflow, the reporter upserts a summary comment (status, tallies, deep link) **from CI using the runner's `GITHUB_TOKEN`** — no GitHub App, no per-tenant install state, no dashboard-side integration (chosen to ship the common case with zero new infra; a GitHub App is the longer-term answer for check-runs / non-Actions runners). New `packages/reporter/src/pr-comment.ts` owns gating/body/posting; `index.ts` tallies per-status counts and invokes the poster after `completeRun` in `onEnd`. Upsert finds a hidden `<!-- wrightful:pr-comment -->` marker among the last 100 comments and PATCHes or POSTs; any error is warned-and-swallowed so a missing comment never fails the suite (the canonical case being a read-only `GITHUB_TOKEN` on a cross-fork PR → 403). Callers must grant `pull-requests: write`. Deferred: per-environment markers (today the single marker means last-run-wins).

## 2026-05-30 — Architecture-deepening sprint

Driven by `docs/reviews/2026-05-30-architecture-deepening-review.html` (findings F01–F94). Organised below by subsystem. Unless noted, each cluster is **behaviour-preserving** (the point was depth/testability, not new features), ran the full gate green (typecheck + dashboard vitest + reporter vitest + `pnpm check`), and added unit tests on the newly-extracted pure surfaces. ### Tenant scope & ingest auth — F03/F07–F14/F36/F65, F66/F84

_(`scoped-data-access-and-run-ownership`, `token-crypto-seam`)_

The highest-severity work. **Scope brand at consumption:** `makeTenantScope(parts)` became the single launder for the two `as Authorized*Id` casts (all of `tenantScopeForUserBySlugs`, `tenantScopeForApiKey`, tenant-context `toScope` funnel through it), and `runScopeWhere(scope)` `(teamId, projectId)` + `runByIdWhere(scope, runId)` `(projectId, runId)` became the blessed predicate builders — both take a `TenantScope`, so the type system now forbids scoping a query from an un-auth-checked id (the brand-consumption invariant is a compile-time guarantee, not a convention). Adopted in `runs-filters-where.ts`, `branches-query.ts`, `ingest.ts`, and the run-scoped routes.

**Credential crypto seam:** `src/lib/token-crypto.ts` concentrates the security-relevant byte/crypto idioms that had drifted into _dangerous_ variants across four files — `sha256Hex`, `base64url{Encode,Decode}` (a guarded loop form vs an unguarded spread that could stack-overflow), constant-time compare (`timingSafeEqualHex` char-wise vs `timingSafeEqualBytes`), and token minting (`mintToken(byteLen, prefix)`, reproducing the old `wrf_`-prefixed API key and unprefixed invite token exactly). `timingSafeEqualHex` now decodes to bytes and delegates to the one constant-time loop. Separately (**F66**) the ingest auth decision was lifted out of `validateApiKey`'s Hono-Context-+-D1 body into a pure `selectMatchingKey(candidates, rawKey)` (hash → constant-time match across same-prefix candidates → reject matched-but-revoked), leaving `validateApiKey` a thin IO wrapper. `+22` tests (`token-crypto.test.ts`, `api-key.test.ts`).

_(`artifact-pipeline`, `f27-artifact-idempotency-index`, `f28-artifact-key-reverse-seam`)_

### Cron watchdog — F80/F81/F82

_(`cron-watchdog`, `f81-bounded-watchdog-sweep`, `f82-silent-noop-finalize`)_

Reworked the stuck-run watchdog from a `createdAt`-keyed, unbounded, serial, sometimes-redundant sweep into a liveness-keyed, budgeted, bounded-concurrency, idempotent one. **F80 (the substantive correctness fix):** the watchdog keyed "stuck?" off `runs.createdAt` (fixed at open), so a live, actively-streaming long suite was force-flipped to `interrupted` while still POSTing. Added `runs.lastActivityAt`, seeded to `createdAt` and bumped to "now" in the _same_ `db.batch` as every ingest write (no extra round-trip); `staleRunFilter` re-keyed to `status='running' AND coalesce(lastActivityAt, createdAt) < cutoff` (coalesce keeps pre-column NULL rows sweepable). The no-delta `/results` branch switched from a read-only `SELECT` to `activityBumpStatement` (an `UPDATE` with the same `.returning()` shape) so a zero-bucket-change flush still registers liveness. **F81 (robustness):** the cron `SELECT`ed every stale run with no `LIMIT` and finalized serially (~2 round-trips each), so a mass-stranding event would self-DoS by exhausting the Workers subrequest/CPU budget. `sweepStaleRuns({cutoffSeconds, limit, now})` owns the budget/concurrency policy and delegates to the pure `drainStaleRuns` orchestrator (bounded-concurrency `Promise.allSettled` waves, tolerant of a stuck run's rejection — the unit-test surface), reduced the cron to a thin adapter, and extracted `chunkBySize` (this caught a latent bug — reusing `chunkByParams` directly would have produced waves of `floor(99/N)`). The backlog drains incrementally across 5-minute invocations. **F82 (efficiency, downgraded from "race"):** a no-op finalize (overlapping pass, or a real `/complete` that won the race) matched 0 rows on the guarded flip but still broadcast a redundant `progress` event; `statementChangedRows` + an opt-in `{requireStatusFlip}` on `reconcileAndBroadcast` make it silent. _Not_ a correctness fix — the broadcast reads status back from the row post-flip, so even on a race it broadcasts the correct terminal.

### Security: origin-safety, CSP & rate-limit coupling — F85/F86/F87/F88/F89/F90/F74

_(`origin-safety`, `csp-unsafe-inline-recorded-decision`, `rate-limit-coupling`)_

**Origin-safety (F88/F90):** pinned the three "attacker-controlled bytes that touch the origin" surfaces with focused tests (the seams already existed). `ansiToHtml` (`src/lib/ansi.ts`) is the only untrusted-text → `dangerouslySetInnerHTML` sink (fed Playwright `errorMessage`/`errorStack` via API-key ingest); tests pin its two-part contract (`Anser.escapeForHtml` first, `use_classes:true` so SGR colours emit `class="ansi-*"`, never a `style=`/`on*=` surface). `SAFE_CONTENT_TYPES` exported as a `ReadonlySet` tied to `buildArtifactHeaders`'s forced `Content-Disposition: attachment`. **CSP decision (F89):** restoring a pre-Void nonce-based CSP is **infeasible on Void 0.8.9** — the framework emits executable inline `<script>` during SSR (deferred-prop resolution `window.__resolveDeferred`, plus the FOUC-killer) with no nonce hook, so a strict `script-src` would break hydration. Recorded the decision via a documented `theme-init-script.ts` seam: `script-src 'unsafe-inline'` is intentionally **load-bearing**, CSP is **not** a second XSS layer, and the ansi sanitiser is the sole guarantee for that sink. Re-entry condition recorded for if Void ever exposes a per-request script nonce. **Rate-limit coupling (F85/F86/F74/F87):** the two ingest route regexes lived byte-for-byte in both the Bearer gate (`02.api-auth.ts`) and the throttle gate (`03.rate-limit.ts`) with two silent drift modes (unbounded per-tenant D1 writes, or a 500 instead of a clean 401) — both now import one pure `isIngestRoute` from `ingest-routes.ts`. Deleted the dead `rateLimit()` per-route factory (zero callers), and exposed `RATE_LIMITER_BINDING_NAMES` as a runtime array so a test asserts the config↔code bijection against `wrangler.jsonc#ratelimits`.
