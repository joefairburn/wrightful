# Void Migration ŌĆö Consolidated Worklog

The Void migration ports the Wrightful dashboard off RedwoodSDK (rwsdk) onto [Void](https://void.cloud) ŌĆö Cloudflare's fullstack Vite plugin + deploy platform ŌĆö in a new parallel package `packages/dashboard-void`. The single biggest architectural shift is collapsing the old two-store data layer (a singleton `ControlDO` + per-team `TenantDO` Durable Objects, queried with Kysely) into a **single D1 database with Drizzle ORM**, swapping physical per-team tenant isolation for logical isolation enforced by `teamId`/`projectId` filters and a branded `AuthorizedProjectId` type. The legacy `packages/dashboard` stays live and untouched; `dashboard-void` is the forward target and (per a 2026-05-23 decision) there will be **no ETL/data migration** ŌĆö the old deployment is abandoned, not cut over.

This document consolidates 16 individual worklogs (2026-05-22 ŌåÆ 2026-05-25). It is the source-of-truth record for the migration; consult it before re-deciding anything in "Key architectural decisions."

## Key architectural decisions

These are durable and must not be re-litigated:

1. **Single D1 over per-team DOs.** Accepts a single-region writer in exchange for cross-team JOINs and one source of truth. Wrightful's write rate (single-digit per team/sec) makes this trivially fine.
2. **Logical tenant isolation, type-enforced.** No DO boundary. Every run-scoped query MUST filter by `teamId AND projectId`. The branded `AuthorizedProjectId` / `AuthorizedTeamId` / `TenantScope` types in `src/lib/scope.ts` force scope checks through the type system before any query runs. `projectId` is denormalized onto every run-scoped child table (`testResults`, `testTags`, `testAnnotations`, `testResultAttempts`, `artifacts`) so the brand enforces scope without runtime joins.
3. **`teamId` denormalized onto `runs`.** Two real justifications under logical isolation: (a) defense-in-depth ŌĆö the `AuthorizedProjectId` brand pairs with a `teamId` filter so a leaked project id can't cross teams; (b) `src/live.ts` does a single-hop `SELECT teamId FROM runs WHERE id = ?` to authorize websocket subscribers. Invariant `runs.teamId == projects.teamId` is set at insert by `tenantScopeFor*` and immutable thereafter.
4. **Better Auth tables stay void-managed.** We declare our own tables but deliberately do NOT declare `user`/`session`/`account`/`verification` ŌĆö void/auth bootstraps them idempotently (`CREATE TABLE IF NOT EXISTS`). Cross-table joins use raw SQL (`db.run(sql\`SELECT ŌĆ” FROM "user"\`)`). Trade-off accepted to avoid index/column collisions.
5. **GitHub login captured into a sibling `userGithubAccounts` table**, not Better Auth `additionalFields`, because we can't extend the auth-owned `account` schema. Mirrored via `databaseHooks.account.{create,update}.after` in `auth.ts`. (The `additionalFields` alternative was repeatedly evaluated and deferred ŌĆö it touches ~12 files.)
6. **Realtime via `void/live`, topic `run:<runId>`.** Replaces the stateful `SyncedStateServer` DO. DO-backed pub-bus with per-topic auth in `onSubscribe` (single SQL `runs Ōŗł memberships`). `runId` is a globally-unique ULID so no team/project disambiguator is needed. The hook shape is `useRunProgress(runId) ŌåÆ { byId, summary }` (accumulator + latest snapshot), not the old `[state, setState]` tuple.
7. **Pages are isomorphic** (Void's Inertia-style pages mode). No `"use client"` boundary ŌĆö all ~54 `ui/` wrappers ported verbatim; only `*.server.ts` files are loaders/actions.
8. **nuqs dropped entirely** in favor of two native hooks (`useSearchParam` shallow / `useNavigatingSearchParam` re-fetch) in `src/lib/use-search-param.ts`. Filter state otherwise goes through `useNavigate()` + `toSearchParams()`.
9. **Drizzle migrations**, generated by `void db generate` into `db/migrations/`, applied on deploy. No rwsdk runtime self-migration. Since nothing has deployed, the initial migration is regenerated in place rather than stacked on.
10. **D1 `db.batch([...])` for atomic multi-statement writes** ŌĆö the correct D1 primitive; no `transactionSync`/`batchTenant` wrappers remain.
11. **Theme tokens are revalued, not renamed.** The redesign remapped `:root`/`.dark` oklch palettes while keeping token names (`--background`, `--accent`, etc.) so the ~57 `ui/` wrappers pick up the new look for free.

## 2026-05-24 ŌĆö Error pages

Added user-facing 404/500 pages + artifact 401 HTML. **404** = `pages/not-found.tsx` (loader sets 404), wired via `void.json#routing.fallbacks` (edge layer, after no-route 404) and `middleware/00.errors.ts`. A catch-all `pages/[...notFound].tsx` was tried first but its `:notFound{.+}` Hono pattern matched Vite dev source URLs and broke HMR ŌĆö fallbacks run at the edge dispatch layer and avoid this. **500** = `pages/oops.tsx`, reached via `c.rewrite("/oops")`.

**`middleware/00.errors.ts`** (runs before context + api-auth) handles two distinct paths: thrown `Response` (loader convention) bubbles past Hono's `compose` and is caught by try/catch around `next()`; thrown `HTTPException` (e.g. `requireAuth`) IS an `Error`, converted to a `Response` by Hono before unwinding, so the middleware inspects `c.res.status` AFTER `await next()`. Behavior: `/api/*` passes through (machine-readable errors preserved); 401ŌåÆredirect `/login`; 404ŌåÆrewrite to 404 page; 5xxŌåÆ`logger.error()` + rewrite `/oops`. **Artifact 401**: `download.ts` returns styled HTML (401) for `Accept: text/html`, plain text for `<img>`/`<video>`/fetch clients.

## 2026-05-25 ŌĆö Artifact XSS hardening, design-bundle port, structural redesign

**Artifact content-type XSS fix (security).** `contentType` was accepted as any non-empty string, flowed to R2 `httpMetadata` and echoed as the download `Content-Type` from the dashboard's own origin with no `Content-Disposition` ŌĆö an API-key holder could register `text/html`, PUT a JS body, and share a signed URL to run script in a victim's session scope. Two-layer defense: (1) **registration allowlist** `src/lib/content-types.ts` (`SAFE_CONTENT_TYPES` + `isSafeContentType`/`safeContentType`), wired into `ArtifactRequestSchema` via `.refine` ŌĆö rejects `text/html`, `image/svg+xml`, `application/javascript`, `application/xhtml+xml`, etc. with 400; (2) **download-time normalization** ŌĆö served `Content-Type` always passed through `safeContentType()` (unsafe ŌåÆ `application/octet-stream`) so legacy rows are covered without a backfill, plus `Content-Disposition: attachment`. Attachment chosen over CSP because the trace viewer cross-origin-fetches artifacts (a CSP wouldn't apply); attachment is honored for top-level navigation, ignored for subresource loads, so `<img>`/`<video>`/trace-viewer paths are unaffected. Upload route also writes the sanitized type to R2. Tests: 91 pass (7 files).
