# 2026-07-12 — Remove unnecessary `useEffect`s (react.dev "You Might Not Need an Effect" pass)

## What changed

Audited all 26 `useEffect` call sites in the monorepo (all in `apps/dashboard`; reporter/e2e have none) against the patterns in
[react.dev/learn/you-might-not-need-an-effect](https://react.dev/learn/you-might-not-need-an-effect), and fixed the seven that
matched an anti-pattern. Everything else (WS room subscriptions, DOM listeners, timers, IntersectionObserver, TanStack cache
invalidation in `run-progress.tsx`, hydration reads) was reviewed and is a legitimate external-system sync — left alone.

## Details

- New file: `apps/dashboard/src/lib/hooks/use-hydrated.ts`. No new dependencies.
- Implemented by four parallel subagents (one per independent file group), then verified centrally.
- The `resolved.testResultId === replay` manual dedup in the trace dialog is now implicit in the query key; switching replay ids
  behaves as before (data goes `undefined` during the fetch, so no stale trace flashes through).
