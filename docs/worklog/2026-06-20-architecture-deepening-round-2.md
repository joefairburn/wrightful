# 2026-06-20 — Architecture deepening (round 2): raw-path coercion seam, child-table tenant predicates, WS room gate factory, CODEOWNERS write seam, artifact-presentation seam, KPI/rate math, offset-pagination, useFeedRoom, retention chunk-cap

## What changed

A second **architecture-deepening** pass (the first was `2026-06-14-architecture-deepening.md`), produced by a fan-out/adversarial-verify review (12 subsystem explorers → per-candidate adversarial verification against the real code → dedup/rank). Of 24 verified candidates, 15 survived and collapsed to **9 ranked deepenings**; all 9 were implemented across **3 file-disjoint waves** with a `vp check` + full-test **integration gate** between each wave.

Every change is **behaviour-preserving** except two deliberate, called-out shifts (see below). The review explicitly excluded the eight seams shipped on 2026-06-14, ADR-0001's realtime-transport decision, the single-store/logical-tenancy decision, and the per-loader numeric-coercion **bug instances** already fixed in `2026-06-20-postgres-migration-review-fixes.md` (this pass deepens the _seam_ behind them, not the instances).

### The 9 deepenings

1. **Enforced numeric-read seam for the raw `runRows` path.** The builder path had a single-owner coercion seam (`numericSql` + the static `aggregate-coercion-guard.test.ts`); the raw `runRows`/`runRow` path had none — every loader hand-typed `cast(… as integer)`/`cast(… as double precision)` with the "int8 is a string" rationale copy-pasted 3-4×. Added a cast-baked aggregate-fragment vocabulary in `src/lib/db/sql-ops.ts` (`intAggExpr`/`numAggExpr` over a private `castAggExpr`, alias-guarded via `assertSqlIdentifier`, emitting `sql.raw` with no bound params), mirroring how `statusCounter`/`percentilePick` bake their own SQL. Adopted at the 7 in-scope count/sum/avg cast sites across `tests.server.ts`, `slowest-tests.server.ts`, `suite-size.server.ts`; end-state SQL byte-identical. `int4` `min`/`max` casts left untouched (already number-safe).

2. **Blessed child-table tenant-predicate family in `scope.ts`.** `scope.ts` owned the runs-table predicate family (`runScopeWhere`/`ciRunsScopeWhere`/`runByIdWhere`/`staleRunFilter`) but the 5 run-scoped **child** tables had no member — the security-load-bearing `eq(<child>.projectId, scope.projectId)` was hand-rolled at ~two dozen sites with two private near-clones. Added a table-agnostic family taking the branded `TenantScope`: `childProjectScopeWhere` + `childByTestResultWhere`/`childByRunWhere`/`childByTestIdWhere`/`childByIdWhere` (new `ColumnRef = Parameters<typeof eq>[0]` keeps it Drizzle-internal-free). Routed the in-scope sites through it; folded the two private clones (`quarantineByTestIdWhere` deleted; `monitorByIdWhere` now delegates to `childByIdWhere`). Behaviour-preserving (emitted `where` identical; Drizzle flattens the nested `and(...)`).

3. **`defineGuardedRoom` — WS room security gate factory.** Both `*.ws.ts` routes duplicated their full `onBeforeConnect` + `onRequest` orchestration verbatim (connect: origin 403 → capacity 429 → authz; publish: method 405 → constant-time secret 403 → server-schema `safeParse` 400 → broadcast → 200), differing only in topic prefix/param/schema pair — so a tweak to the _only_ forged-broadcast gate was an N-file sweep that could silently skip a room. One factory in `room-server.ts` now owns the orchestration; each route collapses to a one-line `defineRoom(defineGuardedRoom({…}))`. Effects (`publicUrl`/`internalSecret` thunk/`authorize`) are **injected** by the routes so `room-server.ts` stays import-light (the pool-workers lane imports it; `@/lib/authz` would drag in `void/db`) and the factory stays a pure function. Status codes, gate order, parse-before-broadcast, and constant-time comparison preserved exactly.

4. **`setCodeownersFile` becomes the single CODEOWNERS write seam.** The repo seam had **zero** production callers and a false doc-comment, while the two real writers had diverged: ingest's `maybeUpdateCodeowners` (unchanged-guard, never null-clears) vs `keys.server.ts` `updateCodeowners` (null-clears, always restamps). Routed both through `setCodeownersFile` and folded both policies into it (trim→null-clear normalization + read-then-skip-write-and-bump unchanged-guard) behind the branded scope. Each adapter keeps only its error mapping (ingest's log-and-continue vs the `codeownersError` flash). **Intended behaviour shift:** the settings writer now skips the `codeownersUpdatedAt` bump on a no-op re-paste.

5. **Server-owned artifact-presentation seam for the test-detail page.** The test-detail loader shipped raw artifact rows _including the internal `r2Key`_ + a token map to the browser, and the page re-implemented the transform inline (its own 5-slot `TYPE_ORDER`, client-side signed-href minting) — drifted from the lib's 4-slot/no-visual copy. Added `loadAttemptArtifactGroups(scope, testResultId, origin)` + a pure `buildAttemptArtifactGroups` (ordering + visual frame-grouping) in `test-artifact-actions.ts` on one shared `TYPE_ORDER` (now with the `visual` slot); a new `SignedArtifact` type **has no `r2Key`**. The loader returns finished groups; the page drops `r2Key`, its `TYPE_ORDER` copy, the token map, and the client token-helper import. `loadFailingArtifactActions` (run-detail) rewired through the same seam. **Security:** the raw R2 object key no longer crosses into SSR HTML. (Per-attempt error bucketing kept in the page.)

6. **KPI/rate math pushed into the loaders + one `rate()` guard.** The `denom <= 0 ? 0 : n/denom*100` idiom recurred at 6 sites and KPI assembly leaked into `.tsx` bodies (untestable; flake-rate was split across the loader/page boundary). New `src/lib/rate.ts` (the only new public module; non-positive denominator → 0, returns 0..100) adopted at all 6 sites. KPI assembly pushed down as internal loader summarizers (`summarizeInsightsKpis`, `summarizeFlakyKpis`) so pages are pure presenters, matching the documented `usage.server` house style. Math relocated, not re-derived.

7. **`resolveOffsetPage` — one offset-pagination math seam.** Three loaders re-open-coded `totalPages`/`currentPage`/`offset`/`fromRow`/`toRow` (drifted: `audit` clamped differently; `tests` had a bespoke refetch-last-page dance), pairing awkwardly with the already-deep `buildPageWindow`/`TablePaginationFooter` rendering seam. Added pure `resolveOffsetPage({total,pageSize,requestedPage,rowCount?})` + `shouldRefetchClampedPage` beside `buildPageWindow`; routed the three full adopters through it. `index.server.ts` is a deliberate partial adopter (its `fromRow`/`toRow` stay in `index.tsx`, coupled to live-row `newCount`). **Intended behaviour shift:** an out-of-range `?page=` on the audit log and slowest-tests now shows the clamped last page instead of "Showing 0 of N" (adopting the tests-catalog behaviour as the single policy).

8. **`useFeedRoom` — one path-generic seed/fold/reconnect-refresh hook.** The no-replay room plumbing (`useSeededState` → `useRoom` → fold through a pure reducer → `requestReconnectRefresh(router.refresh)`) was re-spelled at three sites (`useProjectRoom`, `useRunRoom`, and an inline copy in the monitors list with no reconnect coverage). New `src/realtime/use-feed-room.ts` (`useFeedRoom<P, S>(path, params, seedDeps, seed, fold)`, path-generic against the generated route map) is the single home; all three consumers are thin specializations. Pure reducers untouched; transport untouched (ADR-0001 carve-out).

9. **Retired retention's stale D1-era `ID_DELETE_CHUNK = 98`.** A dead second home for the bound-param ceiling ("under D1's 99-bound-param cap") in a Postgres-only system — the same chunker the team already deleted from ingest's `resolveTestResultIds`. Deleted the constant; `deleteRowsByIds` + the R2-cleanup loop now go through `chunkIdsForInList(ids) = chunkByParams(ids, 1)` (imported from `ingest.ts`, the single param-cap home). Scrubbed residual "D1" wording.

## Cleanup pass — removed run-detail artifact dead weight

A skeptical evaluation of the remaining candidates (do/skip/defer against the real code) returned **6 skip / 1 defer / 0 do** — all failed the deletion test or were below the bar (a JS-coercing `runRows` sibling = dead code competing with the SQL-cast seam; monitor predicate members = 1 eligible site each; `readPeriodCounter` = shallow 2-copy same-file dedup; `runOwnerTenantFormPost` = still 2 adapters, deferred; `emailConfigured()` = bare `Boolean(EMAIL_FROM)` with no normalization; status-bucket routing = would split flaky's correct builder-path `numericSql` idiom). None implemented — shipping a below-bar refactor is a cost, not a win.

The one genuine, evidence-backed finding was the **inverse** of the deferred #5 UX wiring: the run-detail artifact scaffolding was **dead weight** — `loadFailingArtifactActions` was called in `runs/[runId]/index.server.ts` (a DB query + per-failing-test token minting) and the result (`artifactActionsByTestId`) was never destructured by the page; the `ArtifactActions` row component was mounted nowhere. Removed it:

- `runs/[runId]/index.server.ts` — dropped the `loadFailingArtifactActions` call, the `artifactActionsByTestId` field, and the now-unused `origin`.
- `src/lib/test-artifact-actions.ts` — removed `loadFailingArtifactActions`, `FailingTestInput`, and `errorAttempt` (only consumer was the dead loader) + their now-orphaned `and`/`inArray`/`childProjectScopeWhere` imports. The test-detail seam (`loadAttemptArtifactGroups` / `buildAttemptArtifactGroups` / `toVisualArtifactAction` / `SignedArtifact`) is untouched and stays the live path.
- `src/components/artifact-actions.tsx` — removed the unmounted `ArtifactActions` component tree (`ArtifactButton`/`Video`/`Screenshot`/`Trace`/`CopyPromptButton`), keeping only the live type contract (`ArtifactAction`/`VisualDiffGroup`/`VisualDiffFrame`, imported by the test-detail rail + the lib).
- Updated two stale prose comments (`artifacts-rail.tsx`, the test-detail page) that referenced the removed component/helper; extended the module-surface guard test to pin the removed exports.

Behaviour-preserving (the removed loader's output was never rendered) — it eliminates a wasted per-load DB query + token minting on every run-detail view.
