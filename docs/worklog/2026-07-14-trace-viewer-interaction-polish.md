# 2026-07-14 — Trace viewer interaction polish (hover preview, row toggles, source/errors/timeline fixes)

## What changed

Six UX fixes in the custom trace viewer ("Replay"), from dogfooding feedback:

1. **Action rows toggle their accordion.** Expand/collapse of a group row previously required clicking the 14px chevron. Clicking anywhere on a row with children now both selects it and toggles its collapse; the chevron still toggles without selecting (its `stopPropagation` means a chevron click never double-toggles).

2. **Hover-previews the canvas and the Call/Log/Source tabs.** Hovering an action row previews that action in the snapshot pane (including the acted-on element's point marker, which rides along on the snapshot URL) and drives the Call, Log and Source tabs — matching the official viewer, whose workbench renders those views from `highlightedAction || selectedAction`. Hover fires synchronously on `pointerenter` (an initial ~100ms hover-intent debounce was removed after feedback — the official viewer is instant, and the snapshot pane's double-buffered iframes absorb rapid sweeps). On unhover everything reverts to the real (clicked) selection. Architecturally this landed as `activeAction` on the shared `TraceTabProps` contract (alongside `selectedAction`), so each tab declares which action it keys on rather than `DetailTabs` overriding props ad hoc: Call/Log/Source read `activeAction`; Errors/Console/Network/Attachments/Metadata, the timeline overlay, and playback deliberately stay on `selectedAction` so a hover sweep can't yank the Console/Network `scopeToSelected` window, filters, or scroll positions.

3. **Phantom `project#<id>` source tab removed.** Playwright's test runner synthesizes a fixture-pool location `{ file: "project#<projectId>", line: 1, column: 1 }` for project-level `use` option overrides (v1.61.1, `packages/playwright/src/common`, `_buildTestTypePool`). Those synthetic frames land in fixture-setup actions' stacks, so the vendored `collectSources` registered `project#…` as a source file and the Source tab rendered an empty tab for it on every test. A new `isRealSourceFile` helper (`src/trace-viewer/model.ts` — real files contain a path separator) now filters the tab list, the default-file pick (which falls through to the first _real_ file, returning `undefined` if a trace somehow has only synthetic ones), and stack-frame availability (a `project#…` frame renders disabled). The vendored `collectSources` itself is untouched (vendor files stay behavior-identical).

4. **Errors tab count is a red badge.** The Errors tab's count swapped from the muted `text-fg-4` span to the COSS `Badge` (`variant="destructive" size="sm"`). Other tab counts (Console/Network/Attachments) keep the muted span.

5. **Source active line is clearly highlighted.** The target line's `bg-bg-2` (near-invisible against the pane) became `bg-running-soft` plus a 2px inset accent bar (`shadow-[inset_2px_0_0_var(--color-running)]` — an inset shadow, not `border-l`, so the flex content doesn't shift). On a line that is both target and error, the error tint wins the background (twMerge drops the earlier `bg-*`) while the inset bar still marks the target.

6. **Timeline no longer flashes on attempt switches.** The filmstrip thumbnails were keyed by `${frame.sha1}-${i}`; an attempt swap replaces the whole model in place (the workbench stays mounted), so every slot's sha1 changed at once, remounting every `TraceFrameImage` and blanking the strip until the new blobs fetched — while the snapshot pane, which double-buffers, stayed painted. Thumbnails are now keyed by slot index, and `useObjectUrl` gained an opt-in `{ keepPrevious: true }` mode that keeps returning the previous object URL until the new blob resolves, then swaps and revokes the old one (revoke-after-replace; unmount-only cleanup for whatever is displayed; the existing `cancelled` guard means a blob for an abandoned path is never turned into a URL, so nothing leaks). Default-mode behavior and all other call sites are unchanged. The hover-preview card keeps per-hover sha1 keying.
