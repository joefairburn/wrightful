# 2.3 Test ownership / CODEOWNERS

**Tier:** 2 (differentiator) · **Status:** planned

## Decision: both, layered

Manual assignment (`testOwners` table) is source of truth; CODEOWNERS is a derivation from a stored CODEOWNERS file matched against each test's `file`. Manual overrides CODEOWNERS.

## Approach

- **Schema:** `testOwners` (`projectId` FK, `testId`, `owner` opaque label, `source` `"manual"|"codeowners"`, `createdAt`; unique `(projectId, testId, owner)`). Plus `projects.codeownersFile` (text, nullable) + `codeownersUpdatedAt`.
- **`src/lib/codeowners.ts`** — pure `parseCodeowners(text)` + `matchOwners(filePath, rules)` (last-match-wins gitignore-glob semantics), no `void` imports.
- **Ingest (primary):** reporter sends repo CODEOWNERS at `onBegin` (optional `codeowners` string on the open-run payload; extend `OpenRunPayloadSchema` + reporter `types.ts`; `openRun` upserts onto the project). Manual paste in project settings as fallback.
- **Read path:** `resolveTestOwners(scope, testIds)` = union of manual rows + CODEOWNERS-derived (manual wins). Surface owner chips on flaky (add to the `Promise.all` alongside `loadTagsByTestId`), run/test detail headers. Manual "Assign owner" control (owner-gated); repo `src/lib/owners-repo.ts`.

## Verification

- Unit: `codeowners.ts` (glob precedence / last-match).
- Contract test if open-run payload changes.
- Manual: a CODEOWNERS run shows owners + manual override.
