# 1.3 GitHub Checks / Commit Status API

**Tier:** 1 (launch blocker) · **Status:** planned

Post a GitHub check run per run (pass/fail/flaky summary + deep link) to gate PR merges.

## Decision: dashboard-side GitHub App (not reporter-side)

Rationale:

- A CI `GITHUB_TOKEN` is read-only on fork PRs (can't create check runs — the exact case that matters most).
- The dashboard owns the authoritative post-`completeRun` aggregates and the canonical run URL.
- Runs finalized by the watchdog/synthetic path never hit reporter `onEnd`.

The reporter's existing `postPrComment` stays for self-hosters without the App.

## Approach

- **Schema:** `githubInstallations` table (`teamId` FK, `installationId` int unique, `accountLogin`, timestamps); add `runs.githubCheckRunId` (int, nullable) for idempotent POST-then-PATCH. Derive repo from existing `runs.repo`/`runs.commitSha` (no new project config in v1).
- **`routes/api/github/webhook.ts`** — `POST`, verifies `X-Hub-Signature-256` HMAC (reuse `token-crypto` primitives), handles `installation` created/deleted. Self-authenticating, not an ingest route (bypasses `02.api-auth.ts` like artifact download).
- **`src/lib/github-checks.ts`** — `mintInstallationToken` (RS256 JWT via `crypto.subtle` → installation access-token exchange) + `postCheckRun` (POST or PATCH when `githubCheckRunId` set; status→conclusion map in one pure exported fn; `output.summary` mirrors `buildCommentBody`; `details_url` = canonical run URL).
- **Wire:** best-effort `maybePostGithubCheck(scope, runId, summary)` at the end of `completeRun` and `finalizeStaleRun`; resolve installation by `runs.repo` → `accountLogin`; persist returned check-run id; silent no-op when no installation matches.

## Env

`GITHUB_APP_ID`, `GITHUB_APP_PRIVATE_KEY`, `GITHUB_APP_WEBHOOK_SECRET` (all optional; feature off when unset — follow the `githubOAuthEnabled` both-creds-present pattern in `config.ts`).

## Verification

- Unit: status→conclusion map + webhook signature (valid/invalid/missing).
- Integration: drive `completeRun` with stubbed fetch → POST then PATCH on re-complete, no-op without installation.
- Manual: install App on a test repo, open a PR, confirm check + deep link.
