# 2.1 Tag filtering + hierarchical file/suite grouping

**Tier:** 2 (differentiator) · **Status:** planned · **Cheapest win — frontend-mostly**

`testTags` is already populated and the flaky loader already surfaces tags; the tests catalog doesn't expose filtering or grouping. The schema comment at `db/schema.ts` pre-authorizes the index.

## Approach

- **Tag filter** on `tests.{server.ts,tsx}`: add a `tag` query param (multi-select via `MultiComboboxFilter` from `filter-controls.tsx`, driven through `useNavigatingSearchParam`). Add `tagFragment(tags)` to `src/lib/analytics/filters.ts` (injection-safe `EXISTS` correlated subquery against `testTags`, bound params only, same shape as `branchFragment`); apply in both `runPageQuery` and `runAggregateQuery`. Add `loadProjectTags(scope, windowStartSec)` (mirror `loadProjectBranches`).
- **Grouping by file/suite:** purely client-side `groupBy` shallow param (`useSearchParam`, no refetch — rows already loaded). `file` groups on `row.file`; `suite` groups on `row.title.split(" > ").slice(0,-1).join(" > ")`. Collapsible group headers with per-group rollups derived from existing `OutcomeBar` data. Use `SegmentedControl`.
- Optionally extend the flaky page with the same tag filter (it already displays tags).

## Schema

Add `index("testTags_project_tag_idx").on(projectId, tag)` (the schema comment pre-authorizes this).

## Verification

- Unit: `tagFragment`.
- Manual: seed tagged runs, filter + toggle group-by, confirm `count(*) over ()` total reflects the filter and pagination holds.
