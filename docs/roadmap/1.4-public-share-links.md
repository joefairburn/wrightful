# 1.4 Public / shareable run links

**Tier:** 1 (launch blocker) · **Status:** planned

Signed read-only share tokens so a run + its artifacts can be viewed without a session — mirrors the existing artifact-token capability model exactly.

## Approach

- **`src/lib/share-tokens.ts`** — structurally a copy of `artifact-tokens.ts`: `signShareToken({runId, projectId, teamId}, ttl)` / `verifyShareToken` using `token-crypto` HMAC. Recommend a dedicated `SHARE_TOKEN_SECRET` (fallback `?? BETTER_AUTH_SECRET`) so revoke-all is a one-secret rotation. Long-but-bounded TTL (~30d); re-mint 1h artifact tokens on each render. `shareRunHref(token)` owns the `/share/run/:token` URL shape.
- **Revocation (recommended):** a `runShares` table (`tokenHash` via `sha256Hex`, never plaintext — like `apiKeys`; `createdBy`, `expiresAt`, `revokedAt`, denormalized `projectId`/`teamId`). Verify path does a hash lookup + revoked/expiry check.
- **`routes/api/t/[teamSlug]/p/[projectSlug]/runs/[runId]/share.ts`** — `POST` mints (member), `DELETE` revokes. Uses `resolveTenantApiScope` + `runByIdWhere` to confirm the run before signing.
- **`pages/share/run/[token]/{index.tsx,index.server.ts}`** — loader verifies token (expired → styled page reusing `download.ts`'s expired-page pattern), launders the HMAC-verified ids through `makeTenantScope` (**document this as the 4th sanctioned scope producer**), then reuses a refactored `loadRunForView(scope, runId)` (shared with the authed run-detail loader so they can't diverge) + re-minted artifact tokens. Read-only render — no socket, no controls, no team chrome.
- **Middleware:** `middleware/01.context.ts` must let `/share/*` render anonymously (not redirect to `/login`).

## Verification

- Unit: sign/verify round-trip + tamper/expiry/wrong-secret (clone artifact-token tests).
- Integration: anonymous fetch renders run + working artifact links; revoked/expired shows expired page; run A token can't read run B.
- Manual: incognito open.
