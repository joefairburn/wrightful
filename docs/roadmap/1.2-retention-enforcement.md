# 1.2 Retention enforcement (two-axis)

**Tier:** 1 (launch blocker) · **Status:** planned

Enforce the documented two-axis retention: **artifact bytes (R2, ~30d)** and **testResults rows (~90d)** as separate configurable windows. Bytes are the storage-cost axis; rows are the D1-size axis — bytes expire faster, run history persists longer.

## Approach

A retention sweeper cron modeled on `crons/sweep-stuck-runs.ts` → a `src/lib/retention.ts` deep function. Bounded-slice SELECT + bounded-concurrency drain (reuse the generic `drainStaleRuns` orchestrator). Two passes:

- **Artifacts:** select `artifacts` older than `now - retentionArtifactDays` (per project) → delete R2 objects in ≤1000-key pages (`deleteArtifactObjectsByKeys`, sibling to `deleteProjectArtifactObjects`, reuse `DELETE_OBJECTS_MAX_PAGES`) → delete rows.
- **testResults:** delete `testResults` older than `now - retentionTestResultsDays`; FK cascades clean `testResultAttempts`/`testTags`/`testAnnotations`/`artifacts`. Keep `runs` summary rows (aggregate counters live there).

## Schema

- Nullable `teams.retentionArtifactDays` / `teams.retentionTestResultsDays` (null → env default)
- **Add index `artifacts(projectId, createdAt)`** (current artifact indexes don't serve a time-range scan; `testResults_project_createdAt_idx` already exists)

## Env

- `WRIGHTFUL_RETENTION_ARTIFACT_DAYS` (30), `WRIGHTFUL_RETENTION_TEST_RESULTS_DAYS` (90), `WRIGHTFUL_RETENTION_SWEEP_BATCH_SIZE` (200)

## Cron

`crons/sweep-retention.ts`, low-frequency distinct expression (e.g. `0 */6 * * *`).

## UI

Owner-only retention editor on the team general/settings page (`updateRetention` action mirroring `updateGeneral`).

## Verification

- Unit: cutoff filters + drain.
- Integration: seed old/new → assert old gone, new retained, cascades fired, R2 keys deleted (mock storage).
- Manual: tiny windows + seed.
