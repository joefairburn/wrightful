# 2.2 Flaky test quarantine workflow

**Tier:** 2 (differentiator) · **Status:** planned

Turn flaky **detection** into **action**: a per-project quarantine list (keyed by stable `testId`) the reporter pulls at `onBegin`.

## Approach

- **Schema:** `quarantinedTests` (`projectId` FK, `testId`, `reason?`, `mode` default `"skip"` | `"soft"`, `createdBy`, `createdAt`; unique `(projectId, testId)`).
- **`GET /api/runs/quarantine`** — Bearer-authed (already covered by `RUN_INGEST_RE`, no middleware change); `tenantScopeForApiKey` → `{ tests: [{testId, mode, reason}] }`.
- **Reporter** (`packages/reporter`): `quarantine.ts#fetchQuarantine` via `StreamClient` (reuse its retry/version machinery); fetch in parallel with `openRun`; tolerate 404 as "none". **v1 enforcement = demote on the wire** (a reporter can't skip execution — reporters are observe-only): demote a quarantined `failed` to `skipped`/annotated so it doesn't count as a hard failure. Document a companion `test.extend` fixture for true `test.skip()` as a follow-up. Keep `types.ts` ↔ `schemas.ts` in sync (contract canary).
- **UI:** quarantine/unquarantine actions on flaky + test-catalog rows (session-authed `/api/t/*`, owner-gated via `requireOwnerTenantContext`); repo `src/lib/quarantine-repo.ts` mirroring `monitors-repo.ts`; quarantined badge surfaced via a join in `flaky.server.ts` (like `loadTagsByTestId`).

## Verification

- Reporter unit: fetch + demotion.
- Dashboard: repo + cross-tenant scoping returns empty; contract green.
- Manual: e2e via `packages/e2e`.
