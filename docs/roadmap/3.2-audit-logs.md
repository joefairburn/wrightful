# 3.2 Audit logs

**Tier:** 3 (enterprise — defer until a paying customer pulls) · **Status:** planned

Append-only `auditLog` table + one `recordAudit(...)` helper called from each (already concentrated) mutating action.

## Approach

- **Schema:** `auditLog` (`teamId` FK, `projectId?`, `actorUserId` logical FK, `action`, `targetType?`/`targetId?`, `metadata` JSON, `createdAt`; index `(teamId, createdAt)`).
- **`src/lib/audit.ts#recordAudit(c, {...})`** reads actor from `requireAuth(c)`; fire via `c.executionCtx.waitUntil` for non-critical paths — **except deletes, write synchronously before the team cascade removes it**.
- **Instrument the existing actions:** invite mint/revoke/accept, member remove/leave/role-change, key mint/revoke, team rename/delete, project create/delete.
- **Viewer page** `pages/settings/teams/[teamSlug]/audit.{server.ts,tsx}` (owner-only), reverse-chron paginated, actor names via `getUsersByIds`.

## Verification

- Unit: row shape + delete-before-cascade ordering.
- Integration: each action → exactly one correct row.
