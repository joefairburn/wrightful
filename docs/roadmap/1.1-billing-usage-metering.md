# 1.1 Billing / usage metering + quota enforcement

**Tier:** 1 (launch blocker) · **Status:** planned

## Decision

Build Stripe-agnostic metering + quota enforcement now; defer Stripe to a thin later add (a `teams.tier` flip via webhook). The metering layer is identical either way.

## Approach — two-layer meter mirroring the ingest aggregate-delta pattern

- **Live counters** in a new `usageCounters` table keyed `(teamId, periodStart)` (monthly rolling window; `periodStart` derived from `floor(now → month)` so no reset job is needed). Columns: `runsCount`, `testResultsCount`, `artifactBytes`, `artifactCount`, `updatedAt`. Unique index on `(teamId, periodStart)`.
- Increment in the **same `db.batch`** as ingest writes via a new `usageBumpStatement(teamId, period, delta)` (`insert().onConflictDoUpdate()` increment) in `src/lib/usage.ts` — added to the statement arrays in `openRun` (fresh-open branch) and `appendRunResults` (`+results.length`), and in `registerArtifacts` (summed fresh-row bytes only, so idempotent re-registration doesn't double-count).
- **Quota gate** `checkQuota(scope, kind, amount)` read at the cheap entry points (`openRun`, `registerArtifacts`) before writes; returns a discriminated `{ kind: "ok" | "softWarn" | "blocked" }` mapped to a header / 409 by the route (reuse the existing `registerArtifacts` → 413/409 translation pattern).

## Schema

- `teams.tier` (text, default `"free"`)
- `usageCounters` table (above)
- (Stripe-later) nullable `teams.stripeCustomerId` / `teams.stripeSubscriptionId`

## Env

- `WRIGHTFUL_FREE_MONTHLY_RUNS`, `WRIGHTFUL_FREE_MONTHLY_TEST_RESULTS`, `WRIGHTFUL_FREE_ARTIFACT_BYTES`, `WRIGHTFUL_QUOTA_SOFT_WARN_PCT` (default 90)
- A `tierLimits` lookup keyed by `teams.tier` in `src/lib/usage.ts`

## Cron

`crons/rollup-usage.ts` (distinct expression, e.g. `0 3 * * *`) — reconciliation only (recompute counters from authoritative rows for drift correction; period boundary is implicit in the key).

## UI

`pages/settings/teams/[teamSlug]/usage.{server.ts,tsx}` — member-viewable progress bars + a soft-warn banner.

## Verification

- Unit: `usageBumpStatement` delta math + `checkQuota` outcomes (pure).
- Integration: assert counters bump in one batch.
- Manual: set tiny limits, exceed → 409 + 90% soft-warn header.
