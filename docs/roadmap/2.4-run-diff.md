# 2.4 Run-to-run comparison / diff

**Tier:** 2 (differentiator) · **Status:** planned

A pure server-side set-diff over two runs' `testResults` keyed by stable `testId`. **No schema changes.**

## Approach

- **`src/lib/run-diff.ts`**
  - `resolveBaseRun(scope, headRun)` — last `passed` run on same branch before head, served by `runs_project_branch_created_at_idx`.
  - `loadRunTestStatuses(scope, runId)` — `testId, status, durationMs, retryCount` scoped by `projectId`+`runId`.
  - **`diffRuns(base, head)`** — 100% pure, the load-bearing logic: classifies into `newlyFailed` / `newlyPassed` / `stillFailing` / `flakyDeltas` / `addedTests` / `removedTests` + per-test duration deltas.
- **`pages/.../runs/[runId]/diff.{server.ts,tsx}`** — `requireTenantContext`, load head, resolve base (`?base` validated via `runByIdWhere`, else `resolveBaseRun`), `Promise.all` both status loads → render grouped sections (Newly Failed first) with `ui/` table + `status.ts`; base selector via `useSearchParam`. Optional `GET …/diff.ts` JSON route (`Cache-Control: private, max-age=30` like `summary.ts`). Add a "Compare" affordance on the run-detail page.

## Verification

- Unit: `diffRuns` over every bucket incl. identical/no-change.
- Unit: `resolveBaseRun` excludes head + later runs.
- Integration: seed two overlapping-`testId` runs.
- Manual: regression run.
