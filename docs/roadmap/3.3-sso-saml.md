# 3.3 SSO / SAML

**Tier:** 3 (enterprise — defer until a paying customer pulls) · **Status:** deferred (scaffolding built then reverted, 2026-06-14)

> ## ⚠️ Deferred — blocker found, read before re-attempting
>
> A first pass built the full inert scaffolding (env + `ssoEnabled` predicate +
> `teams.ssoDomain` email-domain→team mapping + gated "Continue with SSO" UI),
> but the **plugin itself could not be wired on Cloudflare Workers** and the work
> was reverted (commit after `8c8d895`). The blocker:
>
> - `@better-auth/sso` is **not** bundled in the pinned Better Auth 1.6.11; the
>   version-matched standalone (`@better-auth/sso@1.6.11`) depends on **`samlify`**
>   (→ `xml-crypto`, `@xmldom/xmldom`, `node-rsa`, `node-forge`) — a **Node-native
>   XML/crypto stack that does not bundle on workerd**, and it's pulled in even for
>   OIDC-only use. `@latest` forks a second `better-auth` and must not be used.
> - Auth here is **Void's `defineAuth` wrapper** (`void/auth`), not raw Better
>   Auth. The `plugins` slot does exist (`BetterAuthOptions.plugins`), so the slot
>   is not the blocker — the dependency's Workers-incompatibility is.
>
> **To finish:** use a Workers-safe OIDC path — e.g. `genericOAuth` from
> better-auth's already-bundled `./plugins/generic-oauth` (skips the `samlify`
> baggage), or a vetted SSO plugin whose deps bundle on workerd. Then register it
> conditionally in `auth.ts` (mirror the GitHub gate), re-add the domain-mapping +
> the sign-in hook that calls the join seam, and e2e-test against a local OIDC IdP.
> SAML stays a follow-up on whatever plugin lands.

## Scope: OIDC first via Better Auth's `@better-auth/sso` plugin

Present in the pinned Better Auth 1.6.11 line; supports OIDC + SAML. Enterprise customers overwhelmingly accept OIDC; SAML is a config addition on the same plugin — document as a follow-up.

## Approach

- Register `sso()` in `auth.ts` `plugins: [...]`, conditionally on SSO env presence (mirror the `if (githubClientId && githubClientSecret)` gate so a clean checkout still boots).
- Plugin tables are Better-Auth-owned (NOT in `db/schema.ts`, same boundary as `user`/`session`).
- **Org mapping v1:** email-domain → team via an owner-configured domain on `teams` (reuse the directed-invite seam).
- **UI:** "Continue with SSO" on login/signup gated by an `ssoEnabled` predicate alongside `githubOAuthEnabled`.
- **Env:** optional SSO issuer/client creds (`.secret().optional()`).

## Verification

- Local OIDC test IdP → sign-in creates session + resolves into expected team.
